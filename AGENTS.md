# AGENTS.md

Native macOS browser: Swift 6 + SwiftUI/AppKit, Chromium via CEF, Liquid Glass. Part of the goat ecosystem.

## Stack

- Swift 6, SwiftUI + AppKit. Deployment target macOS 26; uses real Liquid Glass (`glassEffect`, `NSVisualEffectView`, `ConcentricRectangle`).
- Rendering: Chromium Embedded Framework (CEF), version pinned in `scripts/cef-version.txt`. Windowed rendering, one CEF browser per tab.
- Persistence: GRDB (SQLite + FTS5) for history and bookmarks; JSON for sessions; UserDefaults for settings, profiles, and per-site zoom.
- The Xcode project is generated by XcodeGen from `project.yml`. Never edit `GoatBrowser.xcodeproj` — it is generated and gitignored, along with `build/`, `.spm/`, and `ThirdParty/CEF/`.

## Build

```sh
scripts/fetch-cef.sh
scripts/build-wrapper.sh
xcodegen generate
xcodebuild -project GoatBrowser.xcodeproj -scheme GoatBrowser \
  -configuration Debug -destination 'platform=macOS,arch=arm64' \
  SYMROOT="$PWD/build" -clonedSourcePackagesDirPath "$PWD/.spm" build
```

Run `xcodegen generate` after adding or removing a file — new files are not in the project until you do. Incremental builds are reliable; do not wipe DerivedData.

## Run & verify

- Launch: `open "build/Debug/Goat Browser.app"`, or run the binary directly to capture logs:
  `"build/Debug/Goat Browser.app/Contents/MacOS/Goat Browser" >/tmp/goat.log 2>&1 &`
- The real code lives in `Goat Browser.debug.dylib`, not the small launcher executable — grep that dylib when checking a build.
- Chrome DevTools Protocol is on `127.0.0.1:9222` (`curl http://127.0.0.1:9222/json/list`).
- History DB: `~/Library/Application Support/Goat Browser/GoatBrowser.sqlite`.
- Quit with `pkill -TERM`, never `pkill -9` — SIGKILL triggers the macOS "reopen windows?" dialog that blocks the UI.

## Code style

- No comments. Code is self-documenting through naming and small focused types.
- Colors, fonts, spacing, radii, shadows, and motion come from `DS` (`App/DesignSystem.swift`) — never hard-code style values in views.
- Glass is chrome only (sidebar, header, overlays); the web card stays opaque. Never stack glass on glass.
- Models are `@MainActor @Observable final class`. Swift never imports CEF/C++ symbols — it only calls the ObjC `GoatCEF` facade (`Bridge/GoatCEF.h`), which marshals every CEF-thread callback to the main queue.

## Notes

- CEF runs with `--use-mock-keychain --password-store=basic` (`GoatApp::OnBeforeCommandLineProcessing`); without them the keychain prompt blocks page loads.
- Internal pages are served by the `goat://` scheme handler (`Bridge/GoatInternalPages.h`): `goat://newtab`, `goat://error`.
- Keep `BrowserViewModel` actions and the `goat://` router clean — they are the seam for future AI and goat-code integration over CDP.
